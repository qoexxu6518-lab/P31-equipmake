# 游戏系统交互文档制作标准

> 本标准总结自「装备打造系统」交互文档的完整制作流程，用于指导 AI 快速、规范地生成同类文档。

---

## 一、项目定位

**产出物**：一个可直接在浏览器中打开的**单文件 HTML 交互文档**，面向 UI/策划/前端团队评审。

**核心价值**：
- 可操作的高保真原型（非静态截图）
- 新旧 UI 对比，清晰展示改动点
- 交互说明与操作演示一体化，减少沟通成本

---

## 三、index.html 页面三栏布局

```
┌──────────────┬──────────────┬──────────────┐
│   左侧导航栏   │   中央游戏容器  │   右侧旧UI对比  │
│   (380px)     │   (390px)     │   (390px)     │
│              │              │              │
│ · 文档标题    │  可交互的     │ · 面板标题     │
│ · 参考截图按钮 │  游戏界面原型  │ · 旧版截图     │
│ · 界面导航按钮 │  (844px高)    │   (点击可放大)  │
│ · 交互说明区  │              │ · 改动说明     │
│ · 通用规则    │              │              │
└──────────────┴──────────────┴──────────────┘
```

### 3.1 左侧导航栏 (demo-sidebar)

| 区块 | 说明 |
|------|------|
| 文档标题 | 系统名称，居中，金色 (#c8a84e) |
| 参考游戏截图按钮 | 链接到 references.html，可选 |
| 界面导航按钮组 | 每个按钮对应一个界面状态，点击切换中央容器 + 右侧对比 + 下方说明 |
| 交互说明区 | 随导航切换动态显示当前界面的操作说明（HTML 富文本） |
| 通用规则 | 固定显示，适用于所有界面的通用交互规则 |

### 3.2 中央游戏容器 (game-container)

- 固定尺寸 **390×844px**（模拟手机屏幕）
- 包含多个 screen 层（absolute 定位切换显示）
- 需实现可点击的交互（按钮、列表、动画等）
- 对于调用外部通用组件的操作，用 **Toast 提示**代替（如"展示通用道具弹窗"）

### 3.3 右侧旧UI对比面板 (demo-oldui)

- 与游戏容器等高 (844px)
- 显示当前界面对应的旧版截图
- 截图可点击放大（lightbox）
- 下方显示**改动说明**（金色文字，简要描述新旧差异）

## 四、核心 JS 数据结构

```javascript
// 旧UI对比数据：每个界面ID → { 截图路径, 改动说明 }
var DEMO_OLD_UI = {
  idle: { img: 'oldUI1.jpg', note: '改动说明文字...' },
  forging: { img: 'oldUI2.jpg', note: '...' },
  // ...
};

// 交互说明数据：每个界面ID → HTML 富文本
var DEMO_DESCS = {
  idle: '<ul><li>说明1</li><li>说明2</li></ul>',
  forging: '<ul><li>...</li></ul>',
  // ...
};
```

### 4.1 导航同步机制

```javascript
var _demoNav = false; // 防止 demoGo 内部调用覆盖导航状态

function demoGo(id) {
  _demoNav = true;
  // 1. 更新导航按钮高亮
  // 2. 更新交互说明区内容 (DEMO_DESCS)
  // 3. 更新右侧旧UI截图和改动说明 (DEMO_OLD_UI)
  // 4. 切换游戏容器中的 screen
  _demoNav = false;
}

function demoSyncNav(id) {
  if (_demoNav) return; // 被 demoGo 触发时跳过
  // 从游戏交互中自动同步导航状态
}
```

## 六、图片放大弹窗标准实现

两个页面（index.html / references.html）都需要实现的 lightbox 功能：

```
功能要求：
1. 点击图片 → 全屏遮罩显示大图
2. 滚轮无级缩放（基于当前倍率的比例缩放）
3. 左键按住拖拽平移
4. 拖拽中不触发关闭
5. 打开时 body overflow:hidden 屏蔽背景滚动
6. 点击背景/关闭按钮关闭，关闭时重置缩放和位移
```

## 七、制作流程（阶段化推进）

### 阶段 1：基础原型
1. 确认系统功能范围、界面数量、素材清单
2. 搭建三栏布局骨架
3. 实现中央游戏容器的各界面和基本交互
4. 接入素材图片

### 阶段 2：文档化包装
5. 添加左侧导航栏 + 导航按钮
6. 编写各界面的交互说明 (DEMO_DESCS)
7. 实现导航同步机制 (demoGo + demoSyncNav)
8. 添加通用规则区域

### 阶段 3：新旧对比
9. 添加右侧旧UI对比面板
10. 配置 DEMO_OLD_UI 数据（截图 + 改动说明）
11. 实现图片放大弹窗（含缩放 + 拖拽）

### 阶段 4：补充材料
12. 制作参考游戏截图页面（可选）
13. 添加参考截图入口按钮

### 阶段 5：细节研磨
14. 调整字号、间距、颜色等视觉细节
15. 补充/修改交互说明和改动说明文案
16. 处理边界操作（外部组件调用改为 Toast）
17. 最终走查确认

## 八、技术规范

| 项目 | 规范 |
|------|------|
| 文件格式 | 单文件 HTML（CSS/JS 内联），无外部依赖 |
| 字体 | 游戏字体 + 系统字体回退链；侧边栏用微软雅黑 |
| 游戏容器 | 390×844px，模拟手机屏幕比例 |
| 颜色主题 | 深色背景 #0a0a0a，面板 #16161e，金色强调 #c8a84e |
| 外部组件 | 不实现，用 Toast 文字说明代替 |
| 浏览器兼容 | 现代浏览器（Chrome/Edge/Firefox），无需兼容 IE |
| 图片格式 | PNG（UI素材）/ JPG（截图），放在项目根目录 |

## 十、注意事项

1. **渐进式开发**：先完成可运行的基础原型，再逐步添加文档化功能，最后研磨细节
2. **导航同步防冲突**：使用 `_demoNav` 标志位防止 `demoGo` 内部调用覆盖导航状态
3. **外部组件一律 Toast 化**：交互文档不实现外部通用组件，统一用 Toast 说明
4. **图片放大必须支持缩放+拖拽**：方便查看 UI 细节
5. **改动说明要简明扼要**：用"改了什么"而非"怎么改的"的表述
6. **单文件原则**：CSS/JS 全部内联在 HTML 中，确保拷贝即可用，无需搭建环境
